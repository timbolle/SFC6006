Exercice 1 - Analyse de verre (50 points)

Les données

Pour cet exercice, nous allons utiliser des données représentant la composition chimique de morceaux de verre provenant de différentes vitres, fabriquées par deux entreprises. Les données sont présentes dans le fichier data_ex1/glass.csv.

Les colonnes du jeu de données sont les suivantes:

  • company: Indique si le verre provient de l’entreprise A ou B,
  • pane: Indique le numéro de la vitre,
  • fragment: indique le numéro de fragment pour chaque vitre,
  • rep: indique le numéro de réplicat pour chaque mesure de fragment,

Les colonnes suivantes représentes les mesures par LA-ICP-MS des différents composants du verre. Toutes les unités sont en parties par millions (ppm).

Les analyses de 3 fragments de verre inconnus sont également fournis dans le fichier data_ex1/unknown-glass.csv.

L’objectif de cet exercice est de déterminer, si possible, pour chaque fragment:

  • L’entreprise ayant frabriqué la vitre d’origine des fragments
  • La vitre d’origine des fragments

Cet exercice se veut plus libre dans l’approche à suivre.

Exercice 2 - Analyse d’images (50 points)

Objectif

L’objectif de cet exercice est d’explorer différentes approches pour regrouper des images similaires.

Deux principales approches sont envisageables:

  1. A partir de features numeriques extraites des images (ex: couleur, texture, etc.)
  2. A partir de distances entre images basees sur des perceptual hashes (ex: average hash, dhash, phash)

La partie relative au traitement des images (extraction de features, calcul des hashes) est deja realisée et fournie sous forme de tableaux CSV. Votre travail est d’analyser ces donnees pour identifier des groupes d’images similaires, et de comparer les approches.

Différentes images sont fournies pour cet exercice. Ces images sont organisées selon:

  • Les images de base (ex: cat_1, cat_2, dog_1, etc.) représentent des sujets différents.
  • Les images de base transformées (ex: cat_1_crop_large, cat_1_color_jitter, etc.) représentent des variations d’une même image de base.

Les images peuvent être regroupées selon deux niveaux (ground truth):

  1. gt_image: regroupe les images de base et leurs transformations (ex: cat_1 avec cat_1_crop_large, cat_1_color_jitter, etc.)
  2. gt_type: regroupe les images selon un type global (ex: cat_1 et cat_2_crop_large sont du même type “chat”, tandis que dog_1 est d’un type différent “chien”).

Données et fichiers fournis

Les images sont disponibles dans le dossier ZIP data_ex2/image_dataset.zip.

Ces images ont été traitées pour extraire les features et calculer les distances entre les hashs perceptuels. Les fonctions utilisées pour ce traitement sont disponibles dans Utils.R et les résultats sont fournis sous forme de fichiers CSV dans le dossier data_ex2/.

  • data_ex2/features_table.csv : tableau de features extraites des images (ex: couleur moyenne, densité de bords, etc.)
  • data_ex2/hash_dist_matrix_average_hash.csv : matrice de distances pour le hash average
  • data_ex2/hash_dist_matrix_dhash.csv : matrice de distances pour le hash dhash
  • data_ex2/hash_dist_matrix_phash.csv : matrice de distances pour le hash phash
  • data_ex2/hash_dist_long_average_hash.csv : tableau au formatlong de distances pour le hash average
  • data_ex2/hash_dist_long_dhash.csv : tableau au format long de distances pour le hash dhash
  • data_ex2/hash_dist_long_phash.csv : tableau au format long de distances pour le hash phash
  • Examen.Rmd (squelette a completer)

Le tableau de features et les tableaux de distance contiennent deux niveaux de ground truth:

  1. gt_image: même image de base (ex: cat_1 avec ses transformations)
  2. gt_type: même type global (ex: cat_1 et cat_2_crop_large)

Partie 2.1 - Analyse à partir des features

Les features extraites sont les suivantes:

  • Metadonnées géometriques:
    • width, height, aspect_ratio
  • Statistiques globales de couleur (RGB):
    • r_mean, g_mean, b_mean
    • r_sd, g_sd, b_sd
  • Statistiques de luminance (niveau de gris):
    • gray_mean, gray_sd
  • Indicateurs de texture simples:
    • edge_density, gray_entropy
  • Histogrammes normalisés RGB (8 bins par canal):
    • hist_r_1 a hist_r_8
    • hist_g_1 a hist_g_8
    • hist_b_1 a hist_b_8

Le tableau contient également:

  • file (nom du fichier image)
  • gt_image (niveau ground truth image)
  • gt_type (niveau ground truth type)

Les questions suivantes se veulent assez ouvertes. Il est attendu que vous testiez plusieurs approches d’exploration, de clustering et de visualisation.

L’objectif est d’identifier des groupes d’images similaires selon les features, et de comprendre ce que ces groupes capturent (ex: couleur, texture, etc.).

Questions:

À partir de features_table.csv:

  1. Explorez les variables.
  2. Appliquez au moins une méthode de clustering hierarchique.
  3. À l’aide de PCA, essayez de voir si les différents cluster sont visibles. Vous pouvez essayer de colorer selon les catégories ou les groupes mais faites attention au nombre de couleurs
  4. Calculez les matrices de distance et visualisez les selon les ground truth (histogrammes, heatmaps, etc.).
  5. Commentez ce que les clusters capturent (couleur, texture, etc.).
TipIndications (Clustering):

Pour le clustering, vous pouvez utiliser tidyclust, notamment la fonction hier_clust(), qui permet de faire du clustering hierarchique a partir d’une matrice de distance ou d’un tableau de features. Les clusters peuvent être visualisés avec des dendrogrammes. Faites attention qu’il faut généralement indiquer le nombre de clusters à extraire, ce qui n’est pas nécessairement connu dans notre type de problème. Ici, vous pouvez connaitre le nombre de clusters mais ça ne serait pas le cas si nous avions un simple dossier avec un nombre n d’images. Dans ce cas, nous ne sommes pas tant intéressés par les clusters mais plutôt par les distances entre les images.

TipIndications (Distances):

Pour les distances, vous pouvez tester plusieurs mesures (euclidienne, manhattan, etc.) et visualiser les matrices de distance selon les ground truth (ex: heatmap des distances colorees par gt_image ou gt_type). Pour calculer les distances, vous pouvez utiliser la fonction dist() de base R. Vous pouvez également trier pour chaque image, trouver les plus proches, et vérifier si ce sont des images de même gt_image ou gt_type.

N’hésitez pas à illustrer les résultats avec des exemples d’images.

Vous pouvez explorer différentes approches de preprocessing, sélectionner différentes variables ou combinaisons de variables, etc.

Les hash ont été extraits et les distances calculées (distance de Hamming pour les hash binaires).

TipIndications (format des données hash)
  • Les fichiers hash_dist_matrix_*.csv sont au format matrice (wide): première colonne = identifiant image, autres colonnes = distances.
  • Pour un clustering hiérarchique avec tidyclust::hier_clust() (ou hclust()), il faut convertir cette matrice en objet de distance (dist).
  • Les fichiers hash_dist_long_*.csv sont déjà pratiques pour les histogrammes et comparaisons par paires (same_gt_image, same_gt_type).
  • Vous pouvez donc utiliser les deux formats selon la tâche: matrix/wide pour clustering, long pour visualisation/comparaison.

Questions:

  1. Faire du clustering hierarchique directement sur les distances.
  2. Visualiser les distances (heatmap, histogrammes, dendrogrammes).
  3. Comparer les approches (ex: est-ce que les images de même gt_image ont des distances plus faibles que les autres?).

Partie 2.3 - Comparaison et evaluation avec la ground truth

Pour finir, vous devrez comparer les différentes approches explorées jusqu’ici (features vs hashes). Vous avez peut-être déjà fait des comparaisons. Vous pouvez y faire référence ou les reprendre ici.

Pour cela, vous pouvez par exemple produire des histogrammes des distances pour chaque approche, où vous colorez selon si la paire est de même type ou non. N’oubliez pas de discuter des deux types de rapprochements.
